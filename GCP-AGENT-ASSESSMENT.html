4 · Paths forward — mechnet

  • Routing policy is now evidence-backed: quality-critical work stays on sunk local rungs (am4-moe earned top rank on a real task class); gemini rungs are the latency and overflow lane. Current backends.toml tagging (flash routable, pro pin-only) already encodes this — no change needed, the benchmark ratified it.
  • Trial runway healthy: 2.17M / 200M tokens (1%). The binding constraint on GCP experiments is standing infrastructure, not token burn — the runway math should now include deployment-hours, which cloud_spend.json makes visible.
  • Cloud caller: citizen or experiment? Recommendation: keep gcp-adk-test an experiment until (a) Studio ships real auth and (b) a recurring job exists that needs cloud-side session continuity. The moment it becomes standing, it needs: rate limiting, a dedicated profile (the docx's cloud-steward), and a teardown-by-default deployment habit.
  • Manifest gap: the constellation-manifest schema (AM4) has no cost field — if spend should ever roll up constellation-level, that's a schema addition to make there. For now commandcenter's cloud_spend.json is the single home.
  • Identity logging, properly: the "interim identity logging" (project-wide Data-Access audit logs) was the right instinct pointed at the wrong layer — HEARTH's ledger already captures caller identity for everything that crosses the door. GCP-side audit logging only adds value for bypass detection (Vertex-direct calls), which is exactly the sentinel gap already pinned in DECISIONS-PENDING (2026-07-16). Scope audit logs to that question or turn them off.
