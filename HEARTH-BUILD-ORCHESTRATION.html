Frozen contracts (all three build against these verbatim)

1 · Event schema — hearth/contracts/hearth-event.v1.schema.json (H-A commits the file; H-B/H-C code to this shape):

{ "schema": "hearth-event.v1", "event_id": "<uuid4>", "ts": "<ISO-8601 UTC>",
  "caller": { "id": "str", "runner_class": "frontier|local|human", "node": "str" },
  "tool": "str", "args_digest": "sha256:<hex>", "args_preview": "<str, first 400 chars of JSON args>",
  "result_digest": "sha256:<hex>", "ok": true, "error": null,
  "duration_ms": 0, "cost": { "tokens_in": null, "tokens_out": null, "watt_s": null },
  "task_id": null }

2 · Provider contract — each H-B module exposes def get_tools() -> list[Callable]: plain typed python functions with docstrings, JSON-serializable args/returns, no kernel imports. The gateway wraps each callable with auth+provenance+ledger and registers it with FastMCP. Providers stay pure and unit-testable by direct call.

3 · Gateway endpoint — streamable-http at http://127.0.0.1:8710/mcp. Auth: header X-Hearth-Key; registry hearth/etc/callers.json = { "<key>": {"id","runner_class","node"} } (ships with a dev-local test key). Unknown key ⇒ rejected AND the rejection is itself a ledger event.

4 · Ledger API — hearth/kernel/ledger.py: append(event: dict) -> str (validates against schema, appends to hearth/var/ledger/events.ndjson, indexes into index.sqlite), query(caller=None, tool=None, since=None, ok=None) -> list[dict]. Append-only: no update/delete functions exist.

Hearth tests run with that interpreter: & <venv> -m unittest discover -s hearth/tests. Existing repo tests stay on system python: python -m unittest discover -s tests (must remain 177/177 — hearth code must not touch them).

6 · Existing assets to reuse, not reinvent — event append: tools/workflow/append_event.py; projections: tools/workflow/projections.py + project_*.py; guards: tools/workflow/corpus_guard.py, fixture-taint (see tests/workflow/test_fixture_taint.py); target event shape: contracts/workflow-event.schema.json; MCP server exemplar: fleet-worker-node/scripts/worker-mcp-server.py.
