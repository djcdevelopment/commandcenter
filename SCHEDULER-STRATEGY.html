Agent pickup protocol

Any agent, any point — start here
  1. Orient. Read this file top to bottom. Then run the state check command in each step below. Live checks beat the state table — if they disagree, fix the table first. That edit is the living part.
  2. Pick work. Take the first step whose status ≠ DONE and whose dependencies are all DONE. S2 has no dependencies and may always be taken in parallel.
  3. Respect token tiers. Fable = brief/review/judgment. Sonnet = builds. Local qwen via mcp__hearth__local_generate = drafts, triage, summarization. Each step names its tier.
  4. Update on every state change. Edit the state table + changelog in this file, then commit. Doc-only changes go straight to master (standing directive); code changes go branch → tests green → land.
Hard rules — do not improvise around these
  • JS5 stays inert until ADR-0016 is ratified by Derek (H1). Landing actuation code is in scope; turning it on is not.
  • Never dispatch new work to fleet builders without Derek's cue. Everything below is executable locally (OMEN + reads from cc-conductor).
  • No new tooling. This doc, existing scripts, and existing HEARTH door tools are the entire toolkit. If a step seems to need a new dashboard/script/service, it doesn't — write a committed report file instead.
  • Machine lane is mshome/LAN (ADR-0014): conductor = claude@cc-conductor.mshome.net, repo ~/work/commandcenter.

S5 — Regret-trend readout (the evidence for the gate)

TODO
deps: none (JS4 already shipped) · tier: Fable judgment on thin data, script does the math · est: half day · unblocks: H1 → S6
What

hearth/scheduler/hindsight.py (JS4) already replays history through the CP-SAT solver and computes regret (tokens_saved, span_delta_s). Its own docstring names it "the evidence gate that decides whether the shadow scheduler ever earns actuation power." Nobody has written down what it currently says. Produce that readout as a committed report — a document, not a dashboard.

How
  1. Read ADR-0008 first and quote its promotion condition verbatim in the report.
  2. The trend data already exists: since the U3 upgrade (2026-07-04), every 15-minute watchdog patrol tick ledgers a schedule_hindsight regret record. Query the HEARTH ledger for those events first — a week of ticks is the trend series. Supplement with a fresh replay via the door tool mcp__hearth__schedule_hindsight (or directly via hearth.scheduler.hindsight.replay). Record the exact invocation in the report so it's reproducible.
  3. Report: n_runs, aggregate regret, regret split across time windows (is the trend improving?), and one explicit verdict line: GATE: sufficient | insufficient — <one sentence why>.
  4. Commit as docs/REGRET-TREND-2026-07.md.

If there are too few completed runs for a trend, that IS the finding — the verdict becomes "insufficient: need N more scheduled-class runs," which converts H1 from a judgment call into a countdown. Do not pad thin data into a story.

Accepts when: docs/REGRET-TREND-2026-07.md exists on master, contains the reproduction command, and re-running that command reproduces the reported numbers.
Derek: read it — 5 minutes. It exists to make H1 easy.

S6 — The actuation decision (ADR-0016)

GATED · S3 + S5 + H1
deps: S3, S5, H1 · tier: Derek decides, Fable drafts the ADR · est: 1 hr after H1
What

Derek reads the S5 report and ratifies one of: GO (flip actuation on the smallest surface), NO-GO (stays advisory, record why), or WAIT (name the condition that reopens it). Any of the three is a successful outcome — the failure mode is the question continuing to sit unanswered. Record it as docs/adr/0016-scheduler-actuation-decision.md (0016 is next free — verify at write time) and update the register.

If GO

Flip the flag on the narrowest actuation surface available. The first actuated schedule must appear in the ledger with a receipt, and JS4 hindsight must then compare that actuated batch against its own proposal — the loop measuring itself is the finish line of the whole lane.

Accepts when: the ADR file exists on master and the register item is checked off. If GO: additionally one actuated schedule with a ledgered receipt + its hindsight regret record.
Derek: this is the decision. The only hard blocker in the plan.
