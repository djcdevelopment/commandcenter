Two Economies Wind Tunnel

Δ1 — cost_class: metered | sunk

DRAFT SUPERSEDED by Δ1′ — see ECONOMICS-ARCHITECTURE-REVIEW.html

The original objection to Δ1 was framed as a count: don't add a third _influence field alongside policy_influence / capability_influence. That framing doesn't scale and isn't the actual invariant — it was a smell-test standing in for a property that hadn't been named. The real law:

"Any new scheduler decision dimension must explain itself, be replayable, and be independently testable — the same obligations every existing decision dimension already carries. If a proposed dimension satisfies those three properties, its existence is not a constitutional problem. If it can't, it shouldn't exist regardless of how few dimensions currently exist."

This reclassifies the Δ1 problem: the defect isn't "too many influence fields," it's that cost_class: metered | sunk is currently a hand-assigned label with no stated rationale. The real test is whether the dimension can be explained, replayed, and tested — not how many there are.

Δ1′ — cost_class: metered | sunk

Reclassified as a declaration of will: the choice to accept the risk of misclassification in exchange for a simpler, more stable model. The dimension is now fully explainable, replayable, and testable.

Δ2 — physical telemetry channel

APPROVED w/ fix

Correct instance of Amendment 1's residue category. One field fails the test: model_residency: warm | cold is a classification, not a sensor reading — the warm/cold boundary is an authored threshold hiding inside what looks like telemetry. Fix: record last_model_load_event / last_model_unload_event / model_loaded: bool as raw facts; project warm/cold downstream, same as everything else. This is exactly the mistake Clause 1 exists to prevent, caught before it shipped.

Δ3 — learning rate

SPLIT

The metric itself is a clean derivation (sum over existing derived objects, no new state). The risk is downstream: if Δ4's idle-drain dispatcher uses this scalar to rank and select candidates, the metric silently becomes a decision dimension and inherits Amendment 2's obligations without anyone declaring that. Split it — keep learning-rate as a pure human-facing reporting metric, and require any ranking function that consumes it to expose its own terms the way policy_influence already does. Full detail: LEARNING-RATE-CRITIQUE.html.

Δ4 — idle-drain dispatch

APPROVED, conditional

Mechanism is sound and correctly framed as stage-6 autonomy arriving through economics with no new objects. Condition: idleness must become knowledge through events (IdleObserved / IdleEnded), never a polled wall-clock timer inside the scheduler. A timer check is hidden runtime state with no audit trail and violates D18 determinism (not diff-clean, not re-runnable). Evented idle preserves replay, determinism, and auditability in one move.
